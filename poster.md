# ConMate

---

## 1. 문제 정의

공모전 준비는 여러 도구를 오가며 따로따로 진행돼요.

- 공모전은 검색 사이트에서 찾고, 논의는 단체 채팅방에서, 일정은 캘린더에, 계획은 문서로 따로 관리해요.
- 팀장은 지금 어디까지 왔는지 알려면 팀원 한 명 한 명에게 직접 물어봐야 해요.
- 그러다 보니 문제는 항상 마감 직전에야 뒤늦게 발견돼요.

**ConMate는 이 과정을 전부 하나로 모았어요.** 검색 사이트도, 채팅방도, 캘린더도 오갈 필요 없이, 대화형 에이전트와 워크스페이스 하나로 공모전 준비의 전 과정을 관리할 수 있어요.

---

## 2. 서비스 소개

요즘은 공모전 준비에도 AI를 많이 써요. 그런데 대부분 검색이나 요약을 도와주는 정도예요. ConMate는 한발 더 나아가, 스스로 판단하고 필요한 일을 직접 실행하는 AI 에이전트가 팀의 준비 과정에 함께해요.

AI와 함께 공부하고 대화한 내용이, 자동으로 팀 워크스페이스에 저장돼요.

- 대화만으로 내 상황에 맞는 공모전을 탐색하고 추천받을 수 있어요.
- 대화로 주차별 계획을 세우면, 할 일이 팀원에게 자동으로 나눠 배정돼요.
- 오늘 한 일을 말하면, 실행 로그로 남고 관련 할 일이 자동으로 완료 처리돼요.
- AI로 한 번에 팀 현황을 파악할 수 있어요 - 가장 뒤처진 팀원과 이유까지 짚어줘요.

그냥 대화했을 뿐인데, 그 내용이 팀 전체가 보는 워크스페이스(할 일·진행률·리포트)로 자동으로 쌓여요.

---

## 3. 에이전트 동작 방식

한 대화창에서 이 모든 게 가능해요.

| 기능 | 무엇을 하나요 |
|---|---|
| 공모전 탐색·추천 | 관심 분야·참여 형태·목표를 대화로 물어보고, 조건에 맞는 공모전을 찾아 추천해요 |
| 준비 계획 수립 | 주제·마감일·팀원 배경을 바탕으로 주차별 할 일을 만들고 팀원에게 나눠 배정해요 |
| 진행 상황 확인 | 내 진행률, 팀 전체 현황, 다음 주 리스크, 주간 리포트를 물어보면 바로 알려줘요 |
| 실행 기록 | 오늘 한 일을 말하면 로그로 남고, 관련된 할 일이 자동으로 완료 처리돼요 |
| 자유 질문 답변 | "이거 혼자 나가도 돼?", "작년 수상작이 뭐야?" 같은 질문엔 필요한 정보를 직접 찾아본 뒤 답해요 |
| 팀 적합도 평가 | 팀원들의 역량과 공모전 요구사항을 비교해 우리 팀이 얼마나 잘 맞는지 평가해요 |

### 동작 흐름

```
메시지 입력 → 무엇을 원하는지 파악
           → 필요하면 공모전 DB 조회·재검색·웹검색·팀 적합도 평가 등을 직접 실행
           → 답변 생성
           → 워크스페이스(할 일·진행률·리포트)에 반영하고 팀원에게 알림
```

## 5. 구현 및 분석

### 전체 아키텍처

```
[Web] React SPA (Chat / Workspace)
   │ 세션 쿠키
   ▼
[api] FastAPI - 즉시 202+job_id, LLM 직접 호출 없음
   │ Redis LPUSH
   ▼
[Redis] 큐 2개 (recommend / chat)
   │ BRPOP
   ▼
[worker] 에이전트 - 의도 분류 → 도구 호출 → 답변 생성
   │                              │
   ▼                              ▼
App DB (Postgres+pgvector)   Gemini API / 웹검색
(+ 공모전 DB는 읽기 전용 별도 조회)
```

### 대표 기능

- 단계적 추천 - 조건이 덜 모였으면 검색 대신 먼저 되묻는다
- 주차별 계획 자동 배정 - 계획 생성과 동시에 팀원별로 나눠 배정하고 알린다
- 도구 호출형 Q&A - 사실 질문엔 도구로 확인한 뒤 답한다
- 리스크 진단 → 팀장 승인 - 진단은 자동, 반영은 승인 후에만
- 실행 로그 → 자동 완료 처리 - 오늘 한 일을 말하면 기록되고 관련 할 일이 체크된다

## 6. 결론

**차별점**
- 검색을 대신해주는 게 아니라, 스스로 판단하고 필요하면 확인한 뒤 답하는 에이전트
- 도구 호출을 강제하는 규칙으로 환각을 구조적으로 억제
- 되돌리기 어려운 변경은 항상 승인 절차를 거치게 설계

| 구현 한계 | 향후 발전 방향 |
|---|---|
| 일정·회의·인사이트 섹션은 아직 mock | mock 섹션 실 데이터 연동 |
| 데모용 인증 우회 코드가 남아 있음 | 실패한 작업의 재시도(백오프) 자동화 |
