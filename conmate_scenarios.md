## S-01. 공모전 탐색 → 상세 확인 → 선택 → 주제·계획 수립

A가 관심사를 입력해 공모전을 탐색하고, 상세 정보를 확인한 뒤 선택해 워크스페이스와 주차별 계획을 구성하는 흐름

---

### STEP 01. A가 관심사를 입력해 공모전 추천을 요청한다

**행동**
> A: "요즘 브랜드 마케팅이랑 소비자 데이터에 관심 많아. SNS 카피 공모전 나간 적 있고, 이번엔 데이터가 엮인 걸 해보고 싶어. 우리 팀에 맞는 공모전 추천해줘"

**결과**
- AI가 발화에서 관심사(브랜드 마케팅), 경험(SNS 카피), 하고 싶은 것(데이터 분석)을 자동 추출
- 이유가 붙은 공모전 3개 추천
  - ① OO브랜드 소비자 인사이트 공모전 — 마감 5/31 · 500만원 · SNS+데이터 모두 활용
  - ② 대학생 디지털 마케팅 전략 공모전 — 마감 6/15 · 300만원 · 브랜드 전략 중심
  - ③ 소비자 빅데이터 활용 아이디어톤 — 마감 6/20 · 200만원 · 데이터 심화

**케이스들 — 바로 선택하지 않는 경우**

| 사용자 발화 | 시스템 대응 |
|---|---|
| "마감이 너무 촉박한데 다른 거 없어?" | "준비 기간이 얼마나 필요해요?" 역으로 물어서 조건을 좁힌 후 재검색 |
| "이 세 개 비교해줘" | 마감·상금·난이도·팀 적합도 기준으로 비교 표 생성 |
| "우리 팀이 잘 할 수 있을까?" | 팀원 4명 배경·경험과 공모전 요건 매칭 분석 |
| "상금이 더 큰 거 없어?" | 상금 조건 상향 후 DB 재조회. 경쟁률·난이도도 함께 제시 |
| "비슷한 다른 공모전도 보여줘" | 유사 분야·형식 공모전 추가 조회 |

---

### STEP 02. A가 공모전 상세 정보를 확인한다

**행동**
> A: "첫 번째 거 더 자세히 알려줘"

**결과**
- AI가 상세 정보 펼쳐서 제공
  - 주제: 2030 소비자 행동 데이터 활용 마케팅 전략 제안
  - 제출: PPT 20장 이내 + 데이터 분석 보고서
  - 심사: 데이터 활용도 40% · 창의성 30% · 실현가능성 30%
  - 작년 수상: SNS 반응률-구매 전환율 상관관계 분석
  - 예상 준비 기간(4명 기준): 3~4주

**케이스들 — 상세 확인 중 추가 질문**

| 사용자 발화 | 시스템 대응 |
|---|---|
| "작년에 수상한 팀들 어떤 주제로 했어?" | 이전 수상작 주제·접근 방식·차별점 요약. 우리 팀 참고 포인트 함께 제시 |
| "심사위원이 누구야?" | 요강에서 심사위원 정보 파싱. 배경·전공 기반으로 선호 방향 분석 |
| "4명이서 하기에 분량이 너무 많지 않아?" | 제출 요건을 팀원 수로 나눠 예상 작업량 산출 |
| "이거 혼자도 나갈 수 있어?" | 참가 자격(인원 조건) 확인. 개인 참가 가능 여부와 최소·최대 인원 전달 |
| "준비하면 몇 주나 걸릴 것 같아?" | 제출 요건·분량·팀원 수·경험 수준 종합해 예상 기간 산출. 마감 대비 여유 판단 |
| "다른 공모전이랑 비교하면 어때?" | 추천 목록 내 다른 공모전과 1:1 비교. 어느 쪽이 유리한지 분석 |

---

### STEP 03. A가 선택하면 워크스페이스가 자동 생성되고 팀원에게 알림이 발송된다

**행동**
> A: "첫 번째 거로 할게. 팀원들한테 공유하고 워크스페이스 만들어줘"

**결과**
- 워크스페이스 자동 생성
  - 공모전 기본 정보 (주최사·마감·상금·심사기준)
  - 팀원 A · B · C · D 섹션
  - 주제 후보 — 비어있음
  - 실행 로그 — 비어있음
- B·C·D 화면에 알림 발송: "A님이 새 워크스페이스를 열었어요. OO브랜드 소비자 인사이트 공모전 · 마감 5/31"

**케이스들 — 워크스페이스 생성 직후**

| 사용자 발화 | 시스템 대응 |
|---|---|
| "이게 어떤 공모전이야?" (B·C·D) | 각자 채팅창에서 AI가 공모전 요약 설명. 워크스페이스 바로가기 제공 |
| "이 공모전 말고 다른 거 하면 안 돼?" (B) | B 의견을 A에게 알림 전달. A가 팀 투표 열 수 있는 옵션 제시 |
| "섹션 하나 더 추가해줘 — 레퍼런스 모음" | 워크스페이스에 즉시 추가. 팀원 전원에게 업데이트 알림 발송 |

---

### STEP 04. 팀원들이 각자 배경을 입력하면 AI가 주제 후보를 제안한다

**행동**
> B: "2030 건강기능식품 트렌드 공부하고 있어. 데이터 분석 경험 있어"
> C: "인스타그램 광고 반응률 분석 프로젝트 해본 적 있어"
> D: "브랜드 포지셔닝 케이스 많이 봤고, 시각화 잘해"

**결과**
- AI가 4명 배경 종합해 주제 후보 제안
  - 주제 후보 ①: "2030 소비자 SNS 반응 데이터로 보는 OO브랜드 마케팅 전략 제안" (B 트렌드 + C 광고 데이터 + D 시각화)
  - 주제 후보 ②: "건강기능식품 카테고리 브랜드 충성도 강화 전략" (B 데이터 + D 포지셔닝 중심)
- 워크스페이스 주제 후보 섹션에 자동 기록

**케이스들 — 주제 후보 제안 직후**

| 사용자 발화 | 시스템 대응 |
|---|---|
| "주제 1이 너무 광범위해, 좁혀줘" | 좁히는 방향 선택지 제시 (특정 제품군 / 채널 / 행동) 후 재제안 |
| "1번이랑 2번 합치면 어때?" | 두 주제 핵심 요소 결합한 새 후보 생성. 심사 기준 대비 적합도 제시 |
| "이 두 개 말고 다른 각도 없어?" | 경쟁사 비교 / 글로벌 트렌드 / 특정 연령층 집중 등 다른 관점 후보 2개 추가 |
| "이 주제로 수상한 사례 있어?" | 유사 주제 수상 사례 접근 방식·차별점 요약. 차별화 방향 제안 |

---

### STEP 05. 주제 확정 후 AI가 주차별 계획과 팀원별 업무를 자동 배분한다

**행동**
> A: "첫 번째 주제로 확정. 마감 5/31이야. 주차별로 계획 짜고 팀원별로 할 일 나눠줘"

**결과**
- 워크스페이스에 주차별 계획 자동 생성
  - 1주차 (5/3~5/9) 리서치: A-요강 정리 / B-트렌드 데이터 수집 / C-SNS 반응률 분석 / D-경쟁 브랜드 조사
  - 2주차 (5/10~5/16) 교차분석·전략 방향
  - 3주차 (5/17~5/23) 전략 수립·문서화
  - 4주차 (5/24~5/31) 검토·수정·제출
- B·C·D 각자 화면에 이번 주 할 일 자동 반영

**케이스들 — 계획 제안 직후**

| 사용자 발화 | 시스템 대응 |
|---|---|
| "1주차가 너무 빡빡해, 좀 여유 있게 조정해줘" | 1주차 업무 일부를 2주차 초반으로 이동. 마감 역산 기준 이동 가능 여부 먼저 확인 |
| "B가 이번 주 시험이라 일 줄여줘야 해" | B 업무 일부를 다른 팀원에게 재배분하거나 다음 주로 이동. B에게 조정 내용 알림 |
| "계획은 내가 직접 수정할게" | 워크스페이스 계획 섹션 직접 편집 모드 전환. 수정 완료 후 팀원 전체에게 알림 자동 발송 |

## S-02. 공모전 실행 → AI와 함께 진행 → 결과물 워크스페이스 기록

B가 할 일을 확인하고 AI와 함께 실제 공모전 준비를 진행하며, 그 결과물이 팀 워크스페이스에 자동으로 누적되는 흐름

---

### STEP 01. B가 이번 주 할 일을 확인하고 AI와 함께 작업을 시작한다

**행동**
- B가 메인 화면에서 이번 주 할 일 확인
  - 2030 건강기능식품 트렌드 데이터 수집
  - 수집 데이터 정리 및 워크스페이스 업로드
  - D-24 표시
- 할 일 클릭 → AI 채팅창 바로 열림

> B: "2030 건강기능식품 소비 트렌드 데이터 수집해야 하는데 어떻게 시작하면 좋을까?"

**결과**
- AI가 공모전 주제와 연결해서 작업 방향 즉시 제시
  - ① 소비 행동 데이터 (구매 빈도·채널별 비중)
  - ② SNS 반응 패턴 (리뷰·해시태그 분석)
  - ③ 브랜드 인지도 변화
- B 화면에 이번 주 할 일과 D-day 노출
- 팀 전체 진행 현황도 같은 화면에서 확인 가능

**케이스들 — 작업 진행 중**

| 사용자 발화 | 시스템 대응 |
|---|---|
| "이게 우리 공모전 주제랑 어떻게 연결돼?" | 워크스페이스 확정 주제·심사 기준 참조해서 지금 작업이 어느 파트와 연결되는지 설명 |
| "데이터 출처 어디서 찾아?" | 활용 가능한 공개 데이터셋(통계청·식약처·SNS API 등) 목록 제공. 인용 형식으로 정리 |
| "이 분석 방향이 맞아?" | 심사 기준(데이터 활용도 40%) 대비 피드백. 보완 방향 구체적으로 제안 |
| "오늘은 여기까지만 할게" | 지금까지 진행 내용 저장할지 묻는다. "저장해줘" 또는 "다음에 이어서 할게" 옵션 제시 |
| "C한테 이 부분 맡겨도 될까?" | C의 할 일 현황 확인해서 여유 있는지 파악. A에게 업무 이관 요청 알림 보낼지 묻는다 |

---

### STEP 02. B가 작업 결과물을 워크스페이스에 기록한다

**행동**
> B: "오늘 작업한 거 워크스페이스에 저장해줘"

**결과**
- AI가 대화 내용 자동 요약 후 워크스페이스 실행 로그에 기록
  - B · 5/5 · 트렌드 데이터 수집
  - 요약: 구매율 전년比 18%↑. 온라인 비중 67%. 성분 투명성 1순위(54%)
  - 키워드: #건강기능식품 #온라인소비 #성분투명성
- B 화면에서 해당 할 일 자동 완료 처리

**케이스들 — 저장 시**

| 사용자 발화 | 시스템 대응 |
|---|---|
| "이 부분만 저장해줘" | "어느 부분부터 어느 부분까지요?" 확인 후 해당 구간만 추출해서 저장 |
| "저장 말고 C한테만 공유해줘" | 워크스페이스 전체 공개 대신 C 채팅창으로 요약 내용 다이렉트 전송 |
| "지난번에 저장한 거랑 합쳐줘" | 이전 로그와 오늘 내용 병합해서 통합 문서로 워크스페이스 업데이트. 병합 전 미리보기 먼저 제공 |

---

### STEP 03. A가 워크스페이스에서 팀 전체 진행 현황을 확인한다

**행동**
- A가 워크스페이스 실행 로그 섹션을 연다
- 직접 물어보지 않고 팀 전체 현황을 확인한다

**결과**
- 팀 실행 로그에 누가 무엇을 했는지 실시간 표시
  - B · 5/5 — 트렌드 데이터 수집 ✅ 완료
  - C · 아직 없음
  - D · 아직 없음
- 직접 물어보지 않아도 팀 전체가 누가 무엇을 했는지 파악 가능

## S-03. 주간 트래킹 → AI 점검 → 다음 주 계획 재조정

매주 AI가 팀 전체 진행 현황을 자동 집계하고, 미달 항목을 체크해 다음 주 계획을 재조정하는 흐름

---

### STEP 01. 매주 일요일 자정, AI가 자동으로 주간 현황을 집계해 리포트를 생성한다

**행동**
- 별도 행동 없음 — 자동 실행
- AI가 워크스페이스의 할 일 완료 여부와 실행 로그를 읽어 집계

**결과**
- 전원 화면에 알림: "1주차 주간 리포트가 생성됐어요"
- 워크스페이스에 주간 리포트 자동 기록
  - A: 100%
  - B: 50%
  - C: 100%
  - D: 0%
  - 미완료 — B: 데이터 정리 미완료 / D: 1주차 전체 미완료

---

### STEP 02. A가 현황을 확인하고 AI에게 점검을 요청한다

**행동**
> A: "이번 주 현황 어때? D가 하나도 못 했는데 다음 주 계획 괜찮아?"

**결과**
- AI가 단순 현황 나열이 아닌 리스크 진단 제공
  - D의 경쟁 브랜드 조사가 2주차 전략 방향 결정에 필요한 선행 자료. 없으면 2주차 A+D 파트 차단
- 권장 조치 제안
  - ① D의 1주차 과제를 2주차 초반(5/10~5/11)으로 이동
  - ② A+D 전략 회의를 5/12 이후로 조정
- A의 승인 한 번으로 실행 가능

**케이스들 — 리포트 확인 후**

| 사용자 발화 | 시스템 대응 |
|---|---|
