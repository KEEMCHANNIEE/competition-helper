# worker (에이전트) — 개괄 & 동작 설명

이 폴더는 **AI 에이전트**가 사는 곳이다. 사용자의 무거운 요청(추천 만들기, 대화 답하기)을
실제로 처리하는 프로그램이다.

---

## 1. 전체에서 worker의 위치

```
[프론트] → [api] → [Redis 큐] → [worker(여기)] → [App DB] → (프론트가 결과 가져감)
                                     │
                                     ├─ Gemini (AI 모델 호출)
                                     ├─ App DB (대화기록·추천·할일 읽고 씀)
                                     └─ 공모전 DB (공모전 정보 읽기)
```

- worker는 api와 **직접 통신하지 않는다.** api가 Redis 큐에 "작업"을 넣으면, worker가 꺼내 처리한다.
- 결과는 App DB에 저장하고, 프론트는 api를 통해 그 결과를 가져간다.
- worker는 **항상 켜져서 큐를 지켜보는** 프로그램이다.

---

## 2. worker가 처리하는 작업 2종류

큐에서 꺼내는 작업은 두 가지다.

| 작업 | 언제 | 결과 |
|---|---|---|
| **추천** | 사용자가 "추천받기" 했을 때 | 공모전 추천 목록을 DB에 저장 |
| **대화(채팅)** | 사용자가 채팅 메시지를 보냈을 때 | 에이전트 답변을 DB에 저장 |

`main.py`가 큐를 지켜보다가, 추천이면 추천 처리로, 채팅이면 대화 처리로 보낸다.

---

## 3. 에이전트가 동작하는 순서

### (A) 추천 작업 — `agent.run()`
```
1. 큐에서 추천 작업을 꺼낸다 (main.py)
2. 사용자의 관심사·스킬을 DB에서 읽는다
3. 관심사와 비슷한 공모전을 검색한다 (rag.py)
4. 각 공모전마다 "왜 이 사람에게 맞는지" 이유를 AI로 만든다 (llm.py)
5. 추천 목록을 DB에 저장한다
```

### (B) 대화 작업 — `agent.chat()`
대화는 "추천 / 공부 / 계획" 세 가지를 상황에 맞게 한다.

```
1. 큐에서 채팅 작업을 꺼낸다 (main.py)
2. 지금까지의 대화 내용을 DB에서 읽는다 (이전 맥락 기억)
3. 사용자가 뭘 원하는지 판단한다:
     - "공모전 추천해줘"      → 검색 도구로 공모전 찾아 답
     - "이 공모전 평가 기준은?" → 공모전 정보 가져와서 설명 (공부)
     - "계획 짜줘"           → 일정·역할을 만들고 할 일로 저장 (create_tasks)
4. AI로 답변을 만든다 (llm.py)
5. 답변을 DB에 저장한다 → 프론트가 화면에 보여줌
```

핵심: **하나의 에이전트가 대화 안에서 추천·공부·계획을 다 한다.** 무엇을 할지는 사용자의 말에 따라 정해진다.

## 4. 파일별 역할

| 파일 | 하는 일 | 상태 |
|---|---|---|
| `main.py` | 큐에서 작업 꺼내고, 결과를 DB에 저장하는 **흐름 관리** | ✅ 완성 (수정 X) |
| `agent.py` | `run()`=추천, `chat()`=대화. 다른 부품을 불러 **조립·지휘** | 🟨 비어있음(과제) |
| `llm.py` | Gemini에 "글 써줘 / 숫자로 바꿔줘" **요청** | 🟨 비어있음(과제) |
| `rag.py` | 관심사와 **비슷한 공모전 검색** | 🟨 비어있음(과제) |
| `embed_job.py` | 공모전을 **미리 검색 가능하게 준비**(색인) | 🟨 비어있음(과제) |
| `mcp_tools/competitions.py` | 공모전 DB **조건 검색·상세 조회** | 🟨 비어있음(과제) |
| `mcp_tools/semantic.py` | `rag.py`를 도구로 노출 | 🟨 비어있음(과제) |
| `mcp_tools/tasks.py` | 계획을 **할 일로 워크스페이스에 저장**(create_tasks) | 🟨 비어있음(과제) |
| `mcp_tools/registry.py` | 위 도구들을 한데 등록 | 🟨 비어있음(과제) |

- ✅ = 이미 완성된 흐름 관리 코드. **고치지 말 것.**
- 🟨 = AI 담당이 채울 빈칸. 각 파일에 `raise NotImplementedError` 가 들어있다.

---

## 5. 빈칸(과제) 채우는 법

1. `tests/` 안에 각 빈칸의 **검사 코드(실패 테스트)** 가 있다 — "이 함수가 뭘 해야 하는지" 가 적혀 있다.
2. 해당 파일의 `raise NotImplementedError` 를 지우고 내용을 채운다.
3. `uv run pytest`(이 서비스 폴더에서)로 **빨간불 → 초록불** 이 되면 그 과제 완료.

추천 작업 순서:
```
llm.py  →  embed_job.py  →  rag.py  →  mcp_tools/  →  agent.run()  →  agent.chat()
```

> 더 자세한 구현 예시는 `docs/AGENT-GUIDE.md` 참고.
